# 2. 正运动学


上一章我们只控一个关节。这一章把关节串起来——给出 n 个关节角，求末端执行器在世界坐标系下的 6-DoF 位姿。正运动学是逆运动学、步态规划、视觉-动作对齐的前提。

## 本章目标

- 能写出齐次变换矩阵 `T = [R p; 0 1]`
- 能从 URDF / MJCF 描述里读出每个关节的父坐标系与旋转轴
- 能用 Python + NumPy 实现一个 3-DoF 机械臂的 FK，并在 MuJoCo 里验证末端位置误差 < 1e-6
- 能画出雅可比矩阵（为第 3 章逆解做准备）

## 2.1 位姿

位姿 = 位置 + 姿态

先对照下图建立一个直觉:位置只回答"末端原点在哪",姿态只回答"末端坐标轴朝哪",位姿则把两者合在同一个变换里。

<Figure
  src={require('./figs/pose-position-orientation.webp').default}
  caption="位置、姿态与位姿的关系：位置描述末端原点在哪，姿态描述末端坐标轴朝哪，位姿则把两者合在同一个变换里。"
  width={1100}
/>

图里参考坐标系记为 `{B}`,末端坐标系记为 `{E}`,末端原点记为 $O_E$。在继续推公式前,先把三个容易混用的词分开:

- **位置(position)**:只回答"末端原点在哪"。图左侧蓝色虚线向量 $p$ 表示的就是位置,通常写成 ${}^B\!p_E = [x_E, y_E, z_E]^\top$。
- **姿态(orientation)**:只回答"末端坐标轴朝哪"。它描述的是 `{E}` 相对 `{B}` 的朝向,3D 里常用旋转矩阵 $R$、四元数或 `rpy` 表示;平面机械臂里它通常退化成一个角度 $\theta$ 或 $\phi$。
- **位姿(pose)**:位置 + 姿态。它既描述末端原点落在什么位置,也描述末端坐标系的朝向,因此是对刚体状态的完整描述。

这三个概念合在一起,就得到本章后面反复出现的齐次变换矩阵:

$$
{}^B T_E =
\begin{bmatrix}
R & p \\
0 & 1
\end{bmatrix}
\in SE(3)
$$

这里 $p$ 负责"在哪",$R$ 负责"朝哪"。图下半部分画出的,就是这个矩阵的几何意义:它把末端坐标系 `{E}` 中表示的点,变换到参考坐标系 `{B}` 中。对 2D 平面机械臂,姿态进一步退化成一个角度,于是位姿矩阵也退化成常见的 3×3 形式。

本章后面的 3-DoF 平面例子会先把重点放在**位置**上,便于把链式乘法和坐标变换先吃透;到了第 3 章做 IK 时,平面臂的**姿态/朝向**约束会显式写成 $\phi = \theta_1 + \theta_2 + \theta_3$。

## 2.2 关节、连杆与末端执行器

机械臂的运动由两个关键元素组成：**关节**(Joint)和**连杆**(Link)。

- **关节**(Joint)是机械臂的"骨骼连接处"，定义了一个或多个**自由度**(DoF)。最常见的是只贡献一个角度的**旋转关节**(revolute joint)，少数场合会用到**移动关节**(prismatic joint)，状态分别由关节角 $\theta_i$ 或位移 $d_i$ 描述。本章默认讨论旋转关节，因为绝大多数协作机械臂都是 6 个旋转关节串起来的。
- **连杆**(Link)是关节之间的刚体段，本身不会形变，它的位姿完全由两端关节的角度决定。连杆的几何信息（长度、惯量、形状）会进 URDF / MJCF，但 FK 计算只需要它的"父关节到子关节的固定偏置"。
- **末端执行器**(End Effector)是机械臂的"手"，本质上是**最后一个连杆**——可以是夹爪、吸盘、相机或者一根棒。我们之所以单独把它拎出来，是因为下游任务（抓取、装配、视觉对齐）只关心它的位姿，中间连杆的状态只是过程量。

把这三个概念串起来，**正运动学**(Forward Kinematics, FK)就是这样一个函数：

$$
\text{FK}: \theta = (\theta_1, \theta_2, \ldots, \theta_n) \;\longmapsto\; {}^B T_E \in SE(3)
$$

输入是 $n$ 个关节角，输出是末端坐标系相对基座坐标系的位姿。下一节先把"坐标系"这个输出对象讲清楚，再回头看这个映射怎么算。

## 2.3 坐标系

要表达"末端在哪、朝哪"，必须先有参照物——坐标系就是这个参照物。机械臂里常见的坐标系有四种，按"由全局到局部"的顺序排列：

- **世界坐标系**(world frame)：通常记为 `{W}`，是整个仿真或现实场景的全局参考系，地板、桌面、目标物体的位置都在这套坐标系里给。
- **基座坐标系**(base frame)：通常记为 `{B}`，是机械臂根部所在的坐标系。它要么与世界坐标系重合（机械臂固定在原点），要么与世界坐标系之间有一个**已知**的固定变换（机械臂装在移动底盘或工作台上）。
- **连杆坐标系**(link frame)：每个连杆都有一个，记为 `{L_i}`，通常**贴在该连杆的父关节上**。它会随着前序关节转动而在世界里移动，是 FK 链式乘法的中间产物。
- **末端坐标系**(end-effector frame)：记为 `{E}`，贴在末端执行器上，是我们真正关心的输出坐标系。

> 一句话总结：**FK 的本质就是把 `{E}` 中表示的几何量（典型如原点 $O_E$）搬到 `{B}` 或 `{W}` 中**，靠的就是把每对相邻坐标系之间的变换乘起来。下一节先把"一对相邻坐标系之间的变换"写成代数。

## 2.4 齐次变换矩阵

任意刚体变换都可以拆成"先旋转再平移":

$$
{}^A\!p = {}^A_BR\,\,{}^B\!p + {}^A\!p_{B,\text{org}}
$$


机械臂中常用的变换工具是**齐次变换矩阵**，它将位置和姿态合在一起，表示一个刚体的位姿。一个齐次变换矩阵 $T$ 的形式如下：

$$
T = \begin{bmatrix} R & p \\ 0 & 1 \end{bmatrix}
$$

两个关键性质:

- **可以一路乘下去**。$n$ 个关节串起来的末端位姿就是 $T_0^n = T_0^1 T_1^2 \cdots T_{n-1}^n$,这一乘法正是 FK 的全部计算内容。
- **可逆且正交**。$T^{-1} = \begin{bmatrix} R^\top & -R^\top p \\ 0 & 1 \end{bmatrix}$,这在把世界坐标系下的目标搬到末端坐标系时会反复用到。


## 2.5 运动链

把基座、各连杆、末端执行器按"父→子"顺序连成一棵树（机械臂是单链，腿/手是分叉），就得到**运动链**(kinematic chain)。运动链上每条边对应一个关节，每个关节贡献一个齐次变换；FK 就是沿着这条链把所有齐次变换乘起来：

$$
{}^B T_E(\theta) = T_0^1(\theta_1)\,T_1^2(\theta_2)\,\cdots\,T_{n-1}^n(\theta_n)
$$

剩下的问题只有一个：**怎么写出 $T_{i-1}^i(\theta_i)$**？目前主流有两种约定，分别是**DH 参数**和**URDF / MJCF 父-子链式约定**。

### 2.5.1 DH 参数

经典教材（Craig 那本）用 **DH 参数**描述机械臂：每个连杆四个参数 $(a_{i-1}, \alpha_{i-1}, d_i, \theta_i)$，对应一个固定形式的 4×4 矩阵。它的好处是"参数极少、推公式快"；缺点是**不可避免要让连杆坐标系和机械结构错位**，初学者很容易把连杆长度和 $d_i$ 混淆，更尴尬的是 DH 表和 URDF 之间没有自动转换工具，只能逐个关节手算。

### 2.5.2 URDF / MJCF 父-子链

URDF / MJCF 走的是另一条路——**父-子链式约定**：

```
world → base → link1 → link2 → link3 → ... → end_effector
         joint1   joint2   joint3
```

每条 `<joint>` 定义子连杆相对父连杆的**固定偏置 + 轴向**:

```xml
<joint name="joint2" type="revolute" axis="0 0 1">
  <origin xyz="0.3 0 0" rpy="0 0 0"/>
  <parent link="link1"/>
  <child  link="link2"/>
</joint>
```

- 静止部分`origin`(平移+欧拉角)在 URDF 加载时就确定下来,本质是一个常量 $T_\text{offset}$。
- 运动部分:绕 `axis` 旋转 $\theta$ 的增量 $R_\text{joint}(\theta)$。

于是第 $i$ 个关节对应的齐次矩阵总是可以写成:

$$
T_{i-1}^i(\theta_i) = T_\text{offset}\cdot T_\text{joint}(\theta_i)
$$

本章后面的代码都按这个更贴近 URDF 的写法走。好处是**你写的 FK 代码可以直接从 URDF / MJCF 里读参数**，不用再手动换算一遍 DH。两种约定的对照可以记成下面这张表：

| 维度 | DH 参数 | URDF / MJCF 父-子链 |
| --- | --- | --- |
| 每个关节的描述 | 四个参数 $(a, \alpha, d, \theta)$ | 一个 `<origin>` + 一个 `axis` + 一个 $\theta$ |
| 坐标系位置 | 由 DH 规则强制（常与机械结构错位） | 自由放置（一般贴在父关节） |
| 与仿真器衔接 | 需要手算转换 | 加载即用 |
| 适合场景 | 教材推导、解析式 IK | 仿真、真机、可视化 |

## 2.6 平面 3-DoF 手推

作为热身,先考虑最干净的平面 3-DoF 臂——三个连杆长度 $L_1, L_2, L_3$,三个 z 轴旋转关节 $\theta_1,\theta_2,\theta_3$。末端位置是:

$$
\begin{aligned}
x &= L_1\cos\theta_1 + L_2\cos(\theta_1+\theta_2) + L_3\cos(\theta_1+\theta_2+\theta_3)\\
y &= L_1\sin\theta_1 + L_2\sin(\theta_1+\theta_2) + L_3\sin(\theta_1+\theta_2+\theta_3)
\end{aligned}
$$

如果把末端姿态也一起写出来,这个平面例子其实还有一个很干净的结果:

$$
\phi = \theta_1 + \theta_2 + \theta_3
$$

也就是说,这个 3-DoF 平面臂的完整 FK 输出可以理解成 $(x, y, \phi)$。下一章解析 IK 时,这个 $\phi$ 会作为"末端朝向"重新出现。

注意这里关节角是**逐级累加**的——这是下一章解析 IK 能那么干净收敛的关键。

把它写成齐次矩阵形式,方便后面推广到 3D:

$$
T_0^3 = R_z(\theta_1)\,\mathrm{Trans}_x(L_1)\,R_z(\theta_2)\,\mathrm{Trans}_x(L_2)\,R_z(\theta_3)\,\mathrm{Trans}_x(L_3)
$$

把三个 $R_z$ 展开一算,就会发现末端 $(x,y)$ 正是 $T_0^3$ 的第 4 列前两行;这也是所有 URDF 风格 FK 写法的通用套路——**乘出来就直接读平移向量**。

## 2.7 NumPy 实现与 MuJoCo 校验

先定义两个最基础的齐次变换构造器,后面所有 FK 代码都靠它们拼积木:

```python
import numpy as np

def rot_z(theta):
    c, s = np.cos(theta), np.sin(theta)
    T = np.eye(4)
    T[:3, :3] = [[c, -s, 0],
                 [s,  c, 0],
                 [0,  0, 1]]
    return T

def trans(x, y, z):
    T = np.eye(4)
    T[:3, 3] = [x, y, z]
    return T
```

3-DoF 平面臂的 FK:

```python
def fk_planar(thetas, L=(0.3, 0.25, 0.15)):
    t1, t2, t3 = thetas
    T = rot_z(t1) @ trans(L[0], 0, 0) \
        @ rot_z(t2) @ trans(L[1], 0, 0) \
        @ rot_z(t3) @ trans(L[2], 0, 0)
    return T     # 末端位置直接读 T[:3, 3]
```

然后用 MuJoCo 的 `mj_forward` 作为"标准答案"交叉校验——给 100 组随机关节角,比较末端位置的最大误差:

```python
import mujoco

model = mujoco.MjModel.from_xml_path('planar_3dof.xml')
data  = mujoco.MjData(model)
end_id = mujoco.mj_name2id(model, mujoco.mjtObj.mjOBJ_SITE, 'end_site')

max_err = 0.0
for _ in range(100):
    q = np.random.uniform(-np.pi, np.pi, size=3)
    data.qpos[:3] = q
    mujoco.mj_forward(model, data)           # 让 MuJoCo 用 kinematic tree 算
    p_mj = data.site_xpos[end_id]            # 它算的末端位置

    p_ours = fk_planar(q)[:3, 3]             # 我们自己算的末端位置

    max_err = max(max_err, np.linalg.norm(p_mj - p_ours))

print(f'max |p_ours - p_mj| = {max_err:.2e}')
# 期望量级在 1e-14 ~ 1e-12;超过 1e-6 基本是参数/轴向写错了
```

**为什么要花这一步校验?** 因为 FK 是后面所有章节的基石——IK 靠它算残差,雅可比靠它的导数,控制器靠它估末端位置。差 0.5 mm 在第 3 章 `末端画圆`时就是一圈肉眼可见的毛刺。把"和物理引擎完全一致"作为第一次运行的硬指标,后面所有实验才有意义。

## 2.8 雅可比矩阵

FK 把关节角映射到末端位姿。若只关心位置部分,可以写成 $p = f(\theta)$;对时间求导就得到**雅可比矩阵**,它描述"关节角怎么动,末端怎么动":

$$
\dot p = \frac{\partial f}{\partial \theta}\,\dot\theta = J(\theta)\,\dot\theta
$$

对平面 3-DoF 臂,$J$ 是 $2\times 3$(平面位置对 3 个关节求偏导);3D 一般机械臂是 $6\times n$(前 3 行位置、后 3 行旋转,分别叫线速度雅可比和角速度雅可比)。

有两种常用算法:

1. **解析求导**:把 FK 关于每个 $\theta_i$ 求偏导,适合符号化推一次、运行时直接算——速度快,但和具体机器人强绑定。
2. **几何雅可比**:对第 $i$ 列,线速度部分是 $\omega_i \times (p_\text{end} - p_i)$,角速度部分就是 $\omega_i$ (关节轴在世界坐标系下的方向)。**代码写起来通用**,也是下一章 IK 用的那一种。

雅可比是运动学和动力学之间的桥:它一出现,**速度、力、能量**就都能用同一套代数来写——所以第 3 章数值 IK、第 5 章脚端接触力、第 6 章 RL 奖励里都会反复见到它。

## 2.9 实验 1：工作空间散点图

工作空间(reachable workspace)是末端**能到达的所有点**的集合。用 FK 做一次 Monte Carlo 就能把它画出来:

```python
import matplotlib.pyplot as plt

pts = []
for _ in range(20000):
    q = np.random.uniform(-np.pi, np.pi, size=3)
    pts.append(fk_planar(q)[:2, 3])
pts = np.array(pts)

plt.figure(figsize=(5, 5))
plt.scatter(pts[:, 0], pts[:, 1], s=1, alpha=0.3)
plt.axis('equal'); plt.title('3-DoF planar arm workspace')
plt.show()
```

你会看到一个**甜甜圈**形状的区域——外圈是三节完全伸直能到的最远距离 $L_1+L_2+L_3$,内圈是连杆互相折叠能贴到的最近距离。这张图在第 3 章选 IK 目标时会派上用场:**目标点落在甜甜圈外**就是典型的"无解",这时再好的 IK 算法也救不了你。

## 2.10 实验 2：末端位置叠加可视化

MuJoCo 支持在 viewer 里叠加"调试几何",非常适合把 FK 的中间结果可视化出来——比如末端位置、每一节连杆坐标轴、目标点。

```python
import mujoco.viewer

with mujoco.viewer.launch_passive(model, data) as v:
    while v.is_running():
        # 假设我们在 Python 里用自己的 FK 算末端位置
        p = fk_planar(data.qpos[:3])[:3, 3]

        # 清空旧的调试几何再画
        v.user_scn.ngeom = 0
        mujoco.mjv_initGeom(
            v.user_scn.geoms[0],
            type=mujoco.mjtGeom.mjGEOM_SPHERE,
            size=[0.02, 0, 0], pos=p, mat=np.eye(3).flatten(),
            rgba=[1, 0.3, 0.3, 1],
        )
        v.user_scn.ngeom = 1

        mujoco.mj_step(model, data)
        v.sync()
```

这一步看似"只是画个红球",但它实际上是**把 Python 世界里的数学模型和 MuJoCo 世界里的物理模型对上**。如果红球不在机械臂末端上,不是 viewer 的锅——要么坐标系没对齐,要么链式乘法少了一环、多了一环。到第 3 章做 IK 时,**目标点**和**当前末端**都是用这种方式实时叠到场景里的。

## 小结

- **机械臂的组成**：机械臂的运动由两个关键元素组成——**关节**(Joint)和**连杆**(Link)。末端执行器(End Effector)是机械臂的"手"，也是**最后一个连杆**，它的状态由**位置**和**姿态**组成，合起来叫**位姿**(pose)。
- **FK 是什么**：机械臂的正运动学(FK)就是把关节角映射到末端位姿的函数 ${}^B T_E = \text{FK}(\theta)$，位姿用坐标系来描述。机械臂中常见的坐标系有**世界坐标系**、**基座坐标系**、**连杆坐标系**和**末端坐标系**。
- **运动链与齐次变换**：把基座、各连杆、末端按"父→子"串起来就得到运动链；每个关节对应一个"**固定偏置 + 轴向旋转**"的变换，齐次变换矩阵 $T = \begin{bmatrix} R & p \\ 0 & 1 \end{bmatrix}$ 把位置和姿态合在一起，可以**一路乘下去**得到末端位姿。
- **两种描述约定**：DH 参数（教材）和 URDF / MJCF 父-子链式约定（仿真/真机）是描述运动链的两种主流方式，后者更贴近实际机器人描述，本章代码也基于这一约定。
- **从 FK 到雅可比**：齐次矩阵描述末端"在哪、朝哪"，**雅可比矩阵** $J(\theta) = \partial f / \partial \theta$ 描述了"**关节角怎么动，末端怎么动**"的关系，是运动学和动力学之间的桥梁，也是下一章数值 IK 的核心工具。
