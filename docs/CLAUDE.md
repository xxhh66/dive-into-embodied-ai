# 文档写作风格

写本项目 `docs/` 下的章节（尤其 CS123 系列）时遵守以下四条。

## 1. 不写元叙事过渡句

「本节将……」「下一节用 Y 看一下成品」「§4.2 把 X 拆完了，这一节真把 Y 弹出来」之类的承接是无信息量的过渡，删掉让实质内容自己接续。直接讲事实，不预告自己要怎么写。

## 2. 标题严谨且简短

- 用动作-对象短语（"加载 mesh"、"启动 viewer"、"检查关节响应"），避免口语动词（"跑通"、"看什么"、"搞定"）。
- 中文标题字数压在 ~10 字以内。「在 MuJoCo viewer 中加载并查看模型」太长，应写「在 viewer 中查看模型」。

## 3. 标题里只用中英文 + 空格，不放符号

禁止 backtick、`+`、`/`、`→`、`：` / `:` 等。例如：

- ❌ `` `<compiler>` `` + `` `<asset>` ``：加载 mesh
- ✅ compiler 与 asset 加载 mesh
- ❌ fixed → floating：加一行 freejoint
- ✅ 加 freejoint 改 floating

## 4. 跨节引用用 inline

「§4.5 会展开」之类的引用直接放在正文里，不要起一段过渡句独立说明「下面我们将……」。

## 5. 图表引用用「如[图 X](#anchor) 所示」

引用配图时统一写「……如[图 2](#anchor) 所示：」，不要写口语化的「大致是这个样子」「长这样」「就是下面这张图」。表 / 公式同理：「如表 X 所示」「如式 $\eqref{...}$ 所示」。
